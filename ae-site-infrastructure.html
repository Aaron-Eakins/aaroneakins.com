Self-hosted on Linux, Nginx, and Git

This site is also a project. Behind the resume and the write-ups is infrastructure I provision and run myself: a Linux server on DigitalOcean, Nginx, a Git-based deploy, and automated TLS. Building and maintaining it is how I learned the stack hands-on, rather than from a managed host that hides it.

Server and Hosting

The site runs on a DigitalOcean droplet, a small Linux virtual server, on Rocky Linux. Nginx serves the static files. The domain points to the droplet through DNS A records.

Running a plain server instead of a managed one-click host was the point. It meant setting up the operating system, the web server config, and the certificates by hand, which is where the learning is.

Deployment

The site is version-controlled in Git. GitHub Pages serves one copy, and the DigitalOcean droplet serves another through Nginx, both from the same repository. Work happens locally in VS Code and reaches the live site through that Git workflow over SSH.

HTTPS

TLS certificates come from Let's Encrypt, issued and renewed with Certbot. Renewal runs on a schedule, so the certificate does not lapse, and traffic is served over HTTPS.

The Front End

The site itself is hand-written HTML, CSS, and JavaScript on top of Bootstrap. The light and dark themes use CSS custom properties, with the choice saved in the browser and a fallback to the visitor's system setting. A small script handles the sticky header.

Why I Built It This Way

A managed host would have been faster. Running the server, the web server, and the certificates myself was the goal, not a shortcut around it. Maintaining it keeps the Linux, Nginx, and DNS work current instead of theoretical, and it is the groundwork for a self-hosted home lab I'm planning next.
